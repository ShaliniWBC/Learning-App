## Usage Notes

- Replace all `{placeholder}` markers before finalising the report.
- All dates shall use DD-MMM-YYYY format (e.g., 28-Mar-2026).
- RAG statuses shall always include a one-sentence justification.
- Lead with outcomes, not activities. Quantify everything possible.
- Name people and JIRA tickets — never "the team" and "various items".
- "No blockers" and "No decisions required" are valid and should be stated explicitly.
- For the Weekly LT Update, keep total length to one page (screen).
- For the Fortnightly Manager Update, include enough detail for Jeni to act without follow-up questions.
- For the Hawkeye Sponsor Update, the report shall be self-contained — sponsors should not need to ask for context.
- For Sprint Review Summaries, produce one per squad if sprints differ across squads.
- For Monthly Product Health, consolidate data from all sprints within the month.
